Seiso API v1 Reference

HTTP Redirects

Clients should be prepared to handle HTTP 301 Moved Permanently responses, as we may leverage this mechanism to effect URI migrations, especially between major API versions.

A sample request might be the following:

GET /v1/infrastructure-regions HTTP/1.1
Host: seiso.example.com

A sample response might be:

HTTP/1.1 301 Moved Permanently
Location: https://seiso.example.com/v1/regions

To simplify client-side implementation, we recommend implementing redirect-handling code in a cross-cutting fashion so you don't have to implement it for every request.

See the Wikipedia article on HTTP 301 for more information.
